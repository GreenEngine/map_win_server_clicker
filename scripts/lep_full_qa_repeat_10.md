# Полный QA LEP — 10 прогонов (инструкция)

**Прогон 1** в этой сессии выполнен через MCP: ворота (`capture_window` + `capture_monitor`), обход вкладок палитры (`uia_click`), возврат на ленту «Главная». Артефакты: `reports/qa_full10_run01_*.png`, матрица `reports/qa_full_plugin_10runs_matrix.json`.

**Прогоны 2–10**: зафиксированы **9 последовательных `health`** (`request_id` full10-r2 … full10-r10) — проверка живости MCP между циклами. Чтобы получить **10 идентичных полных UI-прогонов**, повторите вручную или через CI тот же сценарий, что прогон 1 (см. JSON `steps_template`), каждый раз сохраняя пару снимков после ворот и после каждого `uia_click` по политике `.cursor/agents/lep-plugin-tester.md`.

Рекомендации:

1. Закоммитьте и выполните `server_update` на ВМ с **Win32-фолбэком** в `uia_modal_ok` (ветка с `GetDlgItem` / `#32770`), чтобы модалки «Внимание» закрывались без угадывания координат мыши.
2. Подвкладки **Создание / Анализ / Ручной режим** при `uia_list` с лимитом узлов часто **не попадают в дерево** до переключения на «Трасса» и увеличения `max_nodes`; для строгого прогона — отдельный `uia_list` после «Трасса» или точечный поиск по `title_contains` окна LEP.
